## Overall Grade: B (70.8/100)

## Executive Summary

OpenClaw v2026.3.31 demonstrates the strongest mechanical constraints and observability in the time series: CI includes dependency boundary enforcement scripts, OTel diagnostics, playwright browser automation, and multi-platform testing. CODEOWNERS is now present with `@openclaw/secops` on security paths. However, `AGENTS.md` has grown to ~286 lines (Encyclopedia anti-pattern), severely impacting documentation and cache-friendly scores. Coverage thresholds and E2E remain configured but not CI-gated. Adversarial verification is absent.

## Anti-Patterns Identified

- **Encyclopedia AGENTS.md**: ~286 lines mixing repo map, architecture law, testing policy, release/security, platform notes, collaboration rules. Directly impacts 1.1, 1.4, 5.5, 6.1.
- **Coverage config without CI enforcement**: `vitest.config.ts` thresholds can drift with CI never running coverage.
- **E2E split from merge path**: High-value flows may regress without required E2E job.

## Improvement Roadmap

### Quick Wins (implement in 1 day)
1. Split `AGENTS.md` into <100-line entry + linked `.agents/skills/` and `docs/` files.
2. Wire `pnpm test:coverage` into CI to enforce the already-configured 70% threshold.
3. Add `pnpm test:e2e` (or minimal smoke) as a required CI check.
4. Create `llms.txt` with curated entry points.

### Strategic Investments (1-4 weeks)
1. Create root `ARCHITECTURE.md` consolidating domain boundaries from scattered docs.
2. Add formal ADR directory for versioned decisions.
3. Implement adversarial verification pipeline with structured evidence format.
4. Add formatter check (`oxfmt --check`) to CI pipeline (currently pre-commit only).
