## 执行摘要

OpenClaw v2026.3.31 在机械约束（分层 CI、`pnpm check` 含类型与 Oxlint、独立 `check-additional` 架构护栏）、安全面标记（`CODEOWNERS` 覆盖 secops 路径）与工具/沙箱信任文档方面达到成熟单仓水准。主要缺口在于：**主 CI 的 `pnpm check` 不含代码格式化（oxfmt）**、**覆盖率阈值未作为默认 CI 必过项**、**Vitest E2E 未纳入主 PR CI 矩阵**，以及缺少 **ADR/设计文档目录** 与 **`llms.txt` 类机器可读索引**，长周期任务的结构化进度工件较弱。

## 改进路线

### 快速见效（约 1 天内）

1. **对齐 CI 与 `pnpm check` 语义**：要么在 `check` job 增加 `pnpm format:check`（或 `oxfmt --check` 与 pre-commit 一致范围），要么将步骤标题改为「types + lint（不含 TS 代码 oxfmt）」，避免虚假绿色。  
2. **在 PR CI 或夜间任务加入 `pnpm test:e2e` 或精选 E2E 子集**，使 4.5 从「有套件」升级为「主链路透传」。  
3. **增加 `llms.txt` 或 `docs/references/` 索引页**，指向核心架构与插件边界文档，提升 5.3。

### 战略投入（1–4 周）

1. **引入 `docs/adr/` 或 `docs/decisions/`**，将网关协议、沙箱与插件边界的关键决策入库，抬高 1.5。  
2. **覆盖率门禁**：在 `ci.yml` 增加可选 job 或在 `main` push 上跑 `pnpm test:coverage` 并解析阈值失败，使 4.3 满分。  
3. **对抗性验证**：为重大网关/安全 PR 增加只读 reviewer 清单或子工作流，输出固定格式证据块（命令、输出摘要、PASS/FAIL），贴近 harness 对抗验证参考。  
4. **技术债工件**：落地 `tech-debt-tracker.json` 或 `docs/QUALITY_SCORE.md`，与 `deadcode`/`dup` 报告链接。
