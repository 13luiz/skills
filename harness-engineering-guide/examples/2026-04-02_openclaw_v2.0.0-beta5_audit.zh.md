# Harness Engineering Audit: OpenClaw v2.0.0-beta5（Clawdis）

**日期**: 2026-04-02  
**Profile**: monorepo | **阶段**: mature  
**生态**: TypeScript/Node.js + Swift + Kotlin（多平台）  
**语言**: 中文（zh）

## 总评等级: C（67.3/100）

## 执行摘要

OpenClaw v2.0.0-beta5（npm 包名 `clawdis`）在机械约束与测试体量上基础扎实：GitHub Actions 在 Node 矩阵上串联 **lint → test → build（含 `tsc`）→ protocol:check**，并辅以 macOS Swift 与 Android Gradle 任务；`AGENTS.md` 共 **78 行**，低于 monorepo 动态阈值 **160 行**（150 + 5×2 workspace 包）。主要短板集中在 **无依赖方向机械门禁、覆盖率阈值未在 CI 中强制执行、缺少 ADR/机器可读 DoD/llms.txt、熵管理与安全护栏（CODEOWNERS、密钥扫描、MCP 配置）不完整**。整体处于「可协作交付，但成熟 harness 仍有关键空洞」区间。

## 改进路线图

### 快速见效（约 1 天内可落地）

1. **新增 `CODEOWNERS`**：为 `src/gateway/**`、`src/web/**`、密钥/协议生成路径配置审查责任人（对应 **8.5**）。  
2. **CI 启用覆盖率门禁**：在 `.github/workflows/ci.yml` 的 Node job 增加 `pnpm test:coverage`（或等价），与 `vitest.config.ts` 中已有阈值对齐（对应 **4.3**）。  
3. **补充根级 `llms.txt`（或 `docs/references/` 索引）**：列出 agent 应优先阅读的文档路径与命令（对应 **5.3**）。  
4. **在 `docker-compose.yml` 增加 `healthcheck`** 并在 README 中写清「健康 = 可连端口/进程就绪」（对应 **7.4**）。

### 战略投入（约 1–4 周）

1. **依赖方向机械规则**：引入 `dependency-cruiser` 或等价，并在 CI 中失败即阻断（**2.5**）。  
2. **机器可读 DoD**：增加 `docs/feature-checklist.json` 或与 `templates/universal/feature-checklist.json` 对齐的跟踪文件（**4.4**）。  
3. **最小 E2E 烟测**：新增 `*.e2e.test.ts` 与 CI job（可仅 nightly），覆盖网关或 CLI 关键路径（**4.5**）。  
4. **对抗式验证流水线**：对协议/安全敏感变更引入只读验证步骤与结构化报告格式（**4.7**）。  
5. **密钥扫描**：在 CI 中增加 `gitleaks`/`trufflehog` 等（填补用户扫描中的缺口）。  
6. **回滚与发布文档**：在 `docs/mac/release.md` 或独立 `docs/rollback.md` 写清版本回退步骤（**8.3**）。
